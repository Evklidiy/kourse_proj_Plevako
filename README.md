1. Обзор проекта
Данный проект представляет собой распределенное приложение на платформе .NET (C#), предназначенное для решения систем линейных алгебраических уравнений (СЛАУ) большой размерности (до 50 000 неизвестных).
Система построена на архитектуре «Master-Worker»:
Master (Ведущий): Узел с графическим интерфейсом, отвечающий за ввод данных, деление матрицы на блоки, координацию вычислений и сбор финального результата.
Worker (Вычислитель): Консольное приложение, выполняющее элементарные преобразования над строками матрицы по командам от Master-узла.
2. Структура решения (Solution)
Проект разделен на логические модули для обеспечения масштабируемости и удобства тестирования:
2.1. Slau.Common (Class Library)
Общая библиотека данных и сетевых протоколов.
MatrixRowTask.cs: Модель данных для передачи строки матрицы и опорного элемента по сети.
NetworkProtocol.cs: Определение команд управления (инициализация, расчет шага, синхронизация, возврат результата).
SerializationHelper.cs: Оптимизированная бинарная сериализация для минимизации сетевых задержек при работе с большими объемами данных.
2.2. Slau.MathEngine (Class Library)
Математическое ядро системы.
GaussJordanSolver.cs: Реализация алгоритма Гаусса-Джордана.
RelaxationSolver.cs: Реализация метода релаксации (локальное решение) для выполнения требования задания по сравнению эффективности распределенного и нераспределенного подходов.
LargeMatrixStorage.cs: Специализированное хранилище на базе double[][] (массив массивов) для обхода ограничений CLR на размер объектов в 2 ГБ, что критично для 50 000 неизвестных.

2.5. Slau.Tests (xUnit/NUnit)

3. Ключевые особенности реализации
Масштабируемость: Поддержка до 50 000 неизвестных. При такой размерности матрица занимает ~18.6 ГБ. Реализовано частичное хранение данных на Воркерах.
Сравнение алгоритмов: В интерфейсе предусмотрена возможность запуска локального метода релаксации для оценки выигрыша в скорости при распределенной обработке.
Ограничения ресурсов: В конфиг-файлах Воркеров реализована возможность ограничения используемой оперативной памяти и количества ядер процессора (согласно условиям задачи).
Отказоустойчивость: Базовая обработка отключения Воркера во время расчета.
4. Инструкция по запуску
Скомпилируйте решение в Visual Studio 2022 (режим Release).
Запустите необходимое количество экземпляров Slau.Worker.exe.
Запустите Slau.Master.exe.
В интерфейсе Мастера укажите IP-адреса Воркеров и размерность системы.
Нажмите "Решить (Распределенный Гаусс-Джордан)".
